Auth
GET /api/auth/login Iniciar sesión
Iniciar sesión en el sistema Parametros
Nombre Descripción
id
[String]		 Representa el identificador o codigo de empleado
pwd
[String]		 Representa la contraseña del perfil del empleado
Respuestas Ejemplo: 

                        
    [
        {
            "tipo":"administrador"
        }
    ]
GET /api/auth/info/{id} Información usuario logueado
Obtiene la información básica del usuario logueado Parametros
Nombre Descripción
{id}
[String]		 Representa el identificador o codigo de empleado
Respuestas Ejemplo: 

                        
    [
        {
            "nombre_usuario": "string",
            "apellidos_usuario": "string",
            "puesto_usuario": "string"
        }
    ]
User-data
GET /api/userdata/all Obtener los usuarios
Obtiene todos los usuarios registrados en el sistema. Parametros
Nombre Descripción
Ninguno
[none]		 No se necesita enviar parametros para esta consulta.
Respuestas Ejemplo: Esta consulta genera diferentes cantidades de registros 

                        
    [
        {
            "id_Usuario":"String",
            "nombre_usuario":"String",
            "puesto_usuario":"String"
        },
        {
            "id_Usuario":"String",
            "nombre_usuario":"String",
            "puesto_usuario":"String"
        }
    ]
GET /api/userdata/info/{id} Obtiene la informacion de un usuario
Obtiene toda la informacion personal asociada a un usuario por medio de su codigo de usuario. Parametros
Nombre Descripción
{id}
[String]		 Representa el identificador o codigo de empleado.
Respuestas Ejemplo: 

                        
    [
        {
            "id_Usuario":"string",
            "tipo_usuario":"string",
            "nombre_usuario":"string",
            "apellidos_usuario":"string",
            "puesto_usuario":"string",
            "telefono_usuario":"string",
            "correo_usuario":"string"
        }
    ]
User-controller
GET /api/usercontroller/generatecode Genera un codigo de usuario
Genera un codigo de usuario para crear un usuario nuevo en el sistema. Parametros
Nombre Descripción
Ninguno
[none]		 No se necesita enviar parametros para esta consulta.
Respuestas Ejemplo: Esta consulta genera diferentes cantidades de registros 

                        
    "EM783491"  Codigo generado por el sistema.
POST /api/usercontroller/createuser Crea un nuevo usuario en el sistema.
Crea un nuevo usuario a partir de la informacion proporcionada por el cliente. Parametros
Nombre Descripción
{array}
[JSON]		 Representa los datos en formato JSON.
Información requerida Ejemplo: 

                        
    [
        {
            "tipousuario":"string",
            "idusuario":"string",
            "claveusuario":"string"
        }
    ]

    Devuelve (Formato JSON)
    "Se realizo 1 registro con exito"
    "Error al crear el registro"
POST /api/usercontroller/createinfouser Crea un nuevo usuario en el sistema.
Guarda toda la informacion personal asociada a un nuevo usuario creado anteriormente. Parametros
Nombre Descripción
{array}
[JSON]		 Representa los datos en formato JSON.
Información requerida Ejemplo: 

                        
    [
        {
            "userid":"string",
            "usertype":"string",
            "username":"string",
            "userlastname":"string",
            "userposition":"string",
            "userphonenumber":"string",
            "useremail":"string"
        }
    ]

    Devuelve (Formato JSON)
    "Se realizo 1 registro con exito"
    "Error al crear el registro"
